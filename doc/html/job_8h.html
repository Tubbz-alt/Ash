[BUILTIN]: Continues active jobs in the background.

Description
This shell builtin is used to continue active jobs (_job) in the background. It accepts the following grammar :
bg -<job-id> ...
To continue the job(s) in the background, it does the following :
  • Checks if the command grammer is valid.
  • If the grammar is valid, then for each job, it calls find_job_id() to find the specified jobs on the active job list (job_list_head) with job ID (_job::id) <job-id> .
  • For each job that is found, it calls continue_job() to continue the job in the background.
Parameters
pThis is a process (_process) . Only its argument vector (_process::argv) and argument count (_process::argc) are of interest to this builtin.
in_fileFile descriptor to the builtin's standard input.
out_fileFile descriptor to the builtin's standard output.
in_fileFile descriptor to the builtin's standard error.
Returns
Returns 0 on success, or -1 on error. Possible reasons for failure are as follows:
  • The grammar of the command is invalid.
  • The grammar of the command is valid, but a specified <job-id> does not refer to a job on the active job list.

[BUILTIN]: List the jobs in the shell's active job list.

Description
This shell builtin is used list the jobs (_job) in the shell's active job list (job_list_head). It accepts the following grammar :
jobs
To list the jobs, it does the following :
  • Calls update() status to update the status of child processes (_process) of the shell.
  • Traverses the active job list, and for each job on the list, it prints its status and then calls print_job_command() to print the command that started the job. An active job can be in one of three states: (1) Running , (2) Stopped , or (3) Done . The function calls job_is_completed(), and job_is_stopped() to determine if an active job is either completed, or stopped, respectively.

Description
This shell builtin is used to send a signal to active jobs. It accepts the following grammar :
kill l | s<signal-name> | n<signal-number> -<job-id> ...
The signal to be sent can be specief by using either the s option, or the
option. The option is used to specified a signal using its symbolic name. The symbolic name must be a case-insensitive string of one of the following patterns:
  • SIGxxx or SIGxxxx (e.g., SIGHUP or SIGKILL, respectively)
  • xxx or xxxx (e.g., HUP or KILL, respectively)
The n option is used to specified the signal number to be sent (e.g., n9 sends signal number 9). The l option is used to list the signals that can be sent. To send the signal, the function does the following :
  • Checks if the command grammer is valid.
  • If the grammar is valid, then for each specified job, it calls find_job_id() to find the job on the active job list (job_list_head)
  • For each job that is found, it calls kill() to send the signal to the job.

int execute_job ( Job *  j)
Parameters
Executesa job.
Description
This function is used to execute a job (_job). It does the following :
  • Performs I/O redirections for the job, if nessasary.
  • Sets up pipes, if nessasary.
  • For each process (_process) in the job, the function checks if it refers to a shell builtin via is_builtin(). If the process refers to a builtin, the builtin is launched. Otherwise, the function forks a child, and the child then calls execute_process() to exec a new program.
  • Cleans up after pipes.
  • After all processes in the job have been launched or executed, the function adds the job to the active job list via add_job_to_list(). In additiion, if Ash_is_interactive is non-zero, it calls wait_for_job() to wait for the job. If Ash_is_interactive is 0 and the job's foreground field (_job::foreground) is non-zero, the function calls put_job_in_foreground() to put the job in the foreground. Otherwise, it calls put_job_in_background() to put the job in the background.
Parameters
jThe job to execute.
Returns
Returns 0 on success, or -1 on failure. Possible reasons for failure are as follows:
  • Performing I/O redirection for the job failed.
  • Setting up pipes failed.
  • A shell builtin failed.
  • Forking a child failed.
  • Cleaning up after pipes (i.e., closing file descriptors) failed.
when any of the events above occurs, the function returns immidiately, and the job is never added to the active job list.

pid_t  pgid,

Executes a process.

Description
This function is used to execute a process (_process). It does the following:
  • If the shell is running interactively (i.e. Ash_is_interactive is non-zero), then this function puts the process in the same process group of the job (_job, _job::pgid) to which it belongs. If the process is the first in the job, then it becomes the group leader of the job.
  • If the shell is running interactively, and foreground is non-zero, then this function makes the group to which the process belong the foreground process group for the session and the controlling terminal (Ash_terminal).
  • When the shell initializes itself via init_shell() , it sets the disposition of all job control stop signals to SIG_IGN (i.e., it ignores those signals). Consequently, each child it creates inherits those signal dispostions. If the shell is running interactively, then this function resets the disposition of those signals to SIG_DFL (i.e. default) .
  • Performs IO redirection, if nessasary.
  • execs the process.
Parameters
pThe process to be executed.
pgidIf this parameter is 0 , then this process becomes the group leader of the job to which it belongs. This happens when the process is the first one in the job.
in_fileThe file descriptor of the file to which standard input for this process should point. If it is not equal to STDIN_FILENO, then standard input for this process is redirected to the file reffered to by this file descriptor.
out_fileThe file descriptor of the file to which standard output for this process should point. If it is not equal to STDOUT_FILENO, then standard output for this process is redirected to the file reffered to by this file descriptor.
err_fileThe file descriptor of the file to which standard error for this process should point. If it is not equal to STD_FILENO, then standard error for this process is redirected to the file reffered to by this file descriptor.
Returns
This function is prototyped to return void , but never returns . If execution of the process (i.e., calling exec()) fails, then the process terminates with err_exit().

void init_shell ( )

Initializes the shell.

Description
The shell calls this function to initialize itself. The function does the following:
  • Assigns STDIN_FILENO to the file descriptor for the shell's terminal (Ash_terminal).
  • Checks if Ash_terminal refers to a terminal devide. If it does, then Ash_is_interactive is set to a non-zero value. Otherwise it is set to 0 .
  • If Ash_is_interactive is non-zero, then the function waits until the shell is in the foreground process group for the session and the controlling terminal (Ash_terminal).
  • Once in the foreground process group of the session, it ignores all job control stop signals by setting thier disposition to SIG_IGN.
  • Puts the shell in its own process group.
  • Makes the shell's process group the foreground process group for the session and controlling terminal.
  • Saves the default terminal modes for the shell in Ash_tmodes
Returns
Returns void .

)

Marks the status of a stopped or terminated child process.

Description
This function is used to mark the status of a stopped or terminated child process (_process). It does the following:
  • Traverses the list of active jobs (job_list_head), and for each job, the function checks its list of processes (#_job::processes) to see if pid matches that of one of its processes. If a match if found, it updates the _process::status field of the correspoinding process.
Parameters
pidThe process ID of the stopped or terminated child process, as returned by waitpid().
statusThe status of the stopped or terminated child process, as returned by waitpid().
Returns
Return 0 on success (i.e., the function successfully marked the status of a process), or -1 on error. The reasons for returning -1 are as follows:
  • pid did not refer to any process in the process list of any job on the active job list.
  • No child process of the shell reported a stopped or termination status (i.e., pid is 0 and errno is ECHILD).
  • Some other error.

  • If the cont parameter is non-zero, it restores the job's saved terminal modes (_job::saved_tmodes), and sends the job the SIGCONT signal.

  • Restores the shell's terminal modes(Ash_tmodes).
